Cadence guard

Healthy probes are capped

Normal worker runs check endpoint_status.last_checked_at first. If the endpoint was checked inside the configured interval, it skips without spending.

Manual override

Force is explicit

Manual debugging can bypass the guard with --force. Without it, even --once respects the 12-hour window.

Recovery

Failures retry sooner

Failing and degraded endpoints retry with exponential backoff: 15 minutes, 30 minutes, 60 minutes, and so on up to the normal 12-hour cap.

Current command behavior
npm run start:worker # Runs once on startup, then wakes on the recovery tick. # Healthy endpoints are checked at most every 12 hours. # Failed endpoints retry with exponential backoff. npm run start:worker -- --once # One-shot check, still cadence/backoff-limited. npm run start:worker -- --once --force # Intentional manual paid probe.
